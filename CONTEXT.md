# Koma — Ubiquitous Language

A glossary of canonical terms for the Koma domain.
This is a glossary, not a spec. Implementation details live in ADRs and code.

## Media types

- **Manga** — Japanese comics. Read right-to-left (unless overridden).
- **Manhwa** — Korean comics / webtoons. Read as a vertical long strip, top-to-bottom.
- **Manhua** — Chinese comics. Read left-to-right.

All three share one reading model — page images, chapter-by-chapter — and differ only by reading direction. Koma carries no anime, no novels, and no torrent media.

## Catalog

- **Catalog** — the metadata backbone: search, browse (by country — JP/KR/CN → manga/manhwa/manhua, trending/popular), covers, alt titles, tags, status, chapter count. Provided by the AniList GraphQL API. Distinct from content (Source) and from tracking (Tracker). AniList is browser-CORS-callable, so the catalog is fetched directly from the web core on every platform — no proxy or Rust fetch needed for it. (COMICK and MangaDex were considered but are Cloudflare/bot-walled from a non-browser client — see ADR 0002.)
- **Title** — a comic entry in the Catalog, identified by its AniList media `id`. Carries cover, synopsis, tags, country of origin, status, year, and chapter count.
- **Chapter** — a unit of a Title. NOT provided by the Catalog (AniList carries no chapter list or contents); chapters come from a Source. A chapter is identified by its number as reported by the Source that served it.
- **Filtered browse** — `browseFiltered()` in `anilist.ts` builds a dynamic AniList GraphQL query from filter state: genre include/exclude, country, status, year range, min chapters. Results cached in IndexedDB. Powers the Categories filter page.

## Sources (content)

- **Source** — a user-declared site the app scrapes for chapter page images. A Source is data, not code: a name, base URL, optional selector overrides, and a CMS preset hint. The app ships three built-in sources: ComicK, MangaPill, and Comick Source API. (MangaDex, Asura Scans, and MangaFire were removed in 2026-07-02 due to Cloudflare walls and unreliable HTML scraping.)
- **Scraper** — the generic engine that, given a Source and a Title+Chapter, fetches the site's chapter page and extracts page image URLs. One engine; no per-site code.
- **Site config** — the declarative description of a Source: URL patterns plus CSS selectors for the chapter list, the page list, and the image list. Imported as a `sources.json` bundle or generated by add-by-URL auto-detect.
- **CMS preset** — a built-in selector template for a common manga CMS. 11 presets exist: `madara`, `mangareader`, `mangadex`, `asura`, `mangafire`, `mangastream`, `genkan`, `wpmanga`, `mangapill`, `comick`, `comick-api`. The fingerprint function detects CMS by HTML markers; first match wins. Used by add-by-URL to auto-configure sources. `mangastream`, `genkan`, and `wpmanga` presets added 2026-07-02.
- **Driver** — a per-source-type adapter that implements `findSeries`, `getChapters`, and `getPages`. Five drivers exist: `html` (CSS selector scraping), `mangadex` (MangaDex API), `comick` (ComicK SSR + REST API), and `comick-api` (unified proxy for 50+ sources).
- **ComicK driver** — scrapes ComicK's SSR HTML pages and REST API. Search uses the `/search?q=...` page (embedded JSON). Chapters use `/api/comics/{slug}/chapter-list?lang=en`. Pages extract from embedded JSON on chapter pages. ComicK chapters carry `group` (scanlation group) and `createdAt` (upload timestamp) for rich display.
- **MangaPill** — a custom manga reader site with fully static HTML (no JS rendering). Uses the HTML driver with the `mangapill` preset. Search at `/search?q=...`, chapters listed in `#chapters` grid on the detail page, page images in `<img class="js-page" data-src="...">` tags served from `cdn.readdetectiveconan.com`. Ships as a built-in source for redundancy alongside ComicK. Added 2026-07-03.
- **Comick Source API driver** — a unified REST API (https://comick-source-api.notaspider.dev) that proxies 50+ manga sources. Search via `POST /api/search`, chapters via `POST /api/chapters`. Page images fall back to the HTML driver. The `apiSourceId` config field selects which upstream source to query. Default changed from `mangaloom` (broken chapters) to `atsumoe` (working) on 2026-07-02. Open-source (MIT) by GooglyBlox.
- **ComicK latest updates** — `comickLatest.ts` scrapes ComicK's home page SSR JSON (`#sv-data`) to extract Latest Updates, Popular Ongoing, Trending (7/30/90 day), Top New Follows, and Completed comics. Used on the Home page as the primary content feed, with AniList as fallback.
- **fetchRaw** — the cross-origin, Referer-spoofing byte-fetch primitive. Rust on desktop; a small proxy on PWA/browser. Returns raw bytes only — all HTML parsing and image handling is in TS. One scraper code path across all platforms.
- **Group mapping** — a community-maintained dataset (https://github.com/GooglyBlox/comick-group-mapping) mapping 761 scanlation group names to their actual websites. Loaded on demand in the Media page to turn group names into clickable links. 741 of 761 groups are mapped.

## Reading & tracking

- **Built-in tracker** — the local source of truth for reading state: per-Title followed flag, per-Chapter read/unread, last-read chapter, reading history, last-read page. Persisted in IndexedDB.
- **Tracker** — an external account (AniList, MAL, MangaUpdates) the built-in tracker syncs progress to. Distinct from Catalog and Source.
- **Progress** — the last-read chapter for a Title, keyed on the AniList media `id` plus the chapter number as reported by the Source that served it. On sync, mapped to the integer chapter number each Tracker expects (AniList/MAL manga progress is integral).
- **Sync** — local-first: the built-in tracker pushes Progress to enabled Trackers on chapter transition; a manual "pull from tracker" imports existing Progress. No bidirectional auto-merge.
- **Reader** — the page-image viewer. Renders manga (RTL paged), manhwa (vertical-webtoon scroll), and manhua (LTR paged) from one component. Direction is derived from the Title's AniList country of origin (KR → vertical-webtoon, CN → LTR, JP → RTL, default RTL), with a per-reader override in every case. Supports keyboard shortcuts (←→↑↓ Space F D G), click-to-advance (tap left/right third), next-chapter preloading, and a floating chapter nav in vertical mode.
- **Library** — the user's followed Titles and reading history, backed by the built-in tracker.

## UI & routes

- **Home** — ComicK-style content feed with Latest Updates (horizontal scroll strip), Trending Now, Popular Ongoing, and Top New sections. All sourced from ComicK's SSR JSON with AniList fallback. Original Browse section (country tabs + sort) preserved below.
- **Search** — AniList search with a top search bar in the nav header (ComicK-style). Quick search from any page via the nav input.
- **Categories** — ComicK-style filter/browse page. Collapsible filter panel with: Origin (All/JP/KR/CN), Sort (Trending/Popular/Latest), Status (Ongoing/Completed/Hiatus/Cancelled), Year range, Min Chapters, and Genre tags (click to include, right-click to exclude with ✓/✕ indicators). Active filter badge count. Results grid with TitleCards.
- **Media** — title detail page with cover, metadata, genres, description, Start Reading/Continue Reading button, Follow toggle, and ComicK-style chapter table (Chap/Uploaded/Group/Actions columns with pagination, goto, read status badges). Chapters auto-resolve from enabled sources — no source picker.
- **Reader** — full-featured reader with direction toggle (RTL/vertical/LTR), image fit (width/screen/original), page filmstrip, keyboard shortcuts, click-to-advance, next-chapter preloading, chapter-end nav, floating chapter nav in vertical mode, and debounced progress persistence.
- **Activity** — reading history timeline grouped by day with relative timestamps.
- **Library** — followed titles grid and reading history list with progress labels.
- **Settings** — Sources (add/import/toggle/remove/reorder/re-check), Cloudflare Unlock (desktop-only, opens a webview to pass Cloudflare challenges and stores cookies for fetch_raw), Trackers (connect/disconnect/enable per tracker), Reader (default direction override), Cache (size readout + clear).

## Cloudflare bypass (desktop only)

- **Cloudflare Unlock** — a feature in Settings that opens a native webview to the target site, lets the Cloudflare JS challenge execute, then extracts cookies (including httpOnly `cf_clearance`) for use in subsequent `fetch_raw` calls. Implemented in `src-tauri/src/cloudflare.rs` with a `CookieStore` (Mutex<HashMap>) managed by Tauri state.
- **How it works** — (1) User enters a URL in Settings → Cloudflare Unlock. (2) A popup webview navigates to the site; the Cloudflare challenge runs in the webview (JS executes). (3) Injected JS polls `document.title` and page content to detect when the challenge passes. (4) Once passed, the webview signals Rust via `cf_challenge_passed` IPC. (5) Rust makes a reqwest request from the same IP (now trusted by Cloudflare) and extracts `Set-Cookie` headers. (6) Cookies are stored in the global `CookieStore` and injected into all subsequent `fetch_raw` calls via the `Cookie` header. (7) If a CAPTCHA is required, the popup stays visible for up to 2 minutes for manual solving.
- **CookieStore** — a global `Mutex<HashMap<String, String>>` keyed by hostname. Populated by `store_cf_cookies` (from JS eval) or by reqwest response header extraction. Read by `fetch_raw` to inject cookies. Persisted to localStorage on the TS side for session continuity.
- **Limitations** — Desktop only (requires native webview). httpOnly cookies (`cf_clearance`) cannot be read by JavaScript; the reqwest follow-up request handles this. Cookies expire based on the site's Cloudflare configuration (typically 30 min to 30 days).

## Provisioning & platforms

- **Add-by-URL** — paste a site URL; the Scraper fingerprints the CMS and creates a Source with no manual selectors.
- **sources.json** — a declarative bundle of Sources (name/url/selectors/preset) the user imports. The PRD's site list takes this shape.
- **Desktop** — the Tauri 2 build; the Rust core performs fetchRaw cross-origin with no CORS and no proxy. Cloudflare unlock available.
- **PWA** — the installable browser build; fetchRaw routes through a small proxy. No Cloudflare unlock.
